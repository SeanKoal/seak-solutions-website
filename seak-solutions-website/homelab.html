THE HOMELAB

Built one host at a time.
Still running.

Everything below actually happened, in order — starting with a thrifted case on a desk and ending with the page you're reading right now, served from a VM on that same machine.

STOP 01 — MAY 2026

The host: Ironhide

Started with a thrifted Fractal Torrent case and built up from secondhand parts — a Ryzen 5 3600, 32GB of DDR4, mixed storage drives, all sourced piece by piece. Proxmox went on as the hypervisor, turning one physical machine into a platform for everything that came after.

PROXMOX VE RYZEN 5 3600 SECONDHAND BUILD
STOP 02

First VM: Ubuntu Server

The first virtual machine went up — a clean Ubuntu Server install, reachable over SSH. Simple on purpose: prove the hypervisor works, prove a VM can be reached remotely, before adding anything more complicated.

UBUNTU SERVER 24.04 SSH
STOP 03

Kali Linux, with a desktop

A Kali VM came next, set up for full remote desktop access rather than just a terminal — useful for the security side of the lab. Getting a graphical desktop to come up reliably over the network turned into its own project, but it now starts clean on boot every time.

KALI LINUX 2026.1 VNC DESKTOP
STOP 04

Real storage: TrueNAS SCALE

A dedicated NAS VM with a physical drive passed straight through to it, formatted into a real storage pool. This is where the lab stopped being just compute and started being a place files actually live.

TRUENAS SCALE DISK PASSTHROUGH ~1.82TB POOL
STOP 05

A network of its own

Every machine and VM got tied together over Tailscale, and the whole lab moved onto its own isolated router and subnet — fully separate from the rest of the household network. From this point on, the lab could be reached securely from anywhere, not just from the same room.

TAILSCALE MESH ISOLATED SUBNET REMOTE ACCESS
STOP 06 — NOW

This website

The newest thing on the network. Hand-coded, served by Nginx on the Ubuntu Server VM, and exposed to the public internet through a Cloudflare Tunnel — no open inbound ports on the router, just an encrypted outbound connection from the server itself. You're looking at the lab's latest tenant.

NGINX HAND-CODED CLOUDFLARE TUNNEL YOU ARE HERE

What it looks like end to end

Same six stops, laid out as the network actually runs today.

No inbound ports opened on the router for this. The Ubuntu Server VM makes an outbound connection to Cloudflare; Cloudflare proxies public traffic to it through that tunnel. Everything else above — the VMs talking to each other, remote management — still stays on the private Tailscale mesh.
